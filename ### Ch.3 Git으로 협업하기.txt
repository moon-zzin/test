# 브랜치
: 작업 내용을 분리해서 독립적으로 개발할 수 있도록 해주는 기능

Git 브랜치는 사실 "커밋을 가리키는 포인터"
새로운 커밋을 하면 그 커밋을 가리키도록 이동

HEAD
: 현재 작업 중인 브랜치를 가리키는 포인터

HEAD가 가리키는 브랜치를 기준으로 커밋 추가

분리된 HEAD 상태 (detached HEAD)
: HEAD가 브랜치를 가리키지 않고 커밋을 직접 가리키는 상태

"분리된 HEAD 상태에서 작업했는데 사라졌어요!"
→ 브랜치를 안 만들고 커밋해서
→ 브랜치 안 만들면 나중에 커밋을 못 찾게 됨
